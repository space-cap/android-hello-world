# 랜덤 점심 메뉴 추천기 (Lunch Menu Random Picker)

**날짜:** 2025-12-09
**상태:** 초안 (Draft)
**작성자:** [사용자 이름]

## 1. 개요 (Summary)
매일 반복되는 "오늘 뭐 먹지?"라는 고민을 해결해 주는 초간단 결정 도우미 앱입니다. 사용자가 미리 등록한 메뉴나 기본 제공되는 메뉴 리스트 중에서 하나를 무작위로 추첨하여 재미있게 보여줍니다.

## 2. 배경 및 문제 의식 (Background & Problem)
*   **결정 피로 (Decision Fatigue):** 현대인은 너무 많은 선택지 속에서 사소한 결정에 에너지를 낭비합니다. 특히 점심 메뉴 선정은 매일 반복되는 스트레스입니다.
*   **기존 앱의 복잡함:** 배달 앱이나 맛집 추천 앱은 정보가 너무 많아 오히려 선택을 지연시킵니다.
*   **재미 요소 부재:** 단순한 사다리 타기나 제비뽑기는 시각적인 즐거움이 부족합니다.

## 3. 제안 내용 (Proposal)

### 3.1 핵심 컨셉
*   **Zero Friction:** 앱 실행 -> 버튼 클릭 -> 결과 확인. (3초 컷)
*   **Fun:** 슬롯머신이나 룰렛이 돌아가는 듯한 시각적 연출로 기대감 조성.

### 3.2 주요 기능
1.  **랜덤 추첨 (The Picker)**
    *   화면 중앙의 거대한 '추천받기' 버튼.
    *   버튼을 누르면 메뉴 텍스트가 빠르게 바뀌다가 서서히 멈추는 애니메이션 (Slot Machine / Roulette effect).
    *   결과 화면에서 "다시 뽑기" 또는 "확정(지도 보기 등)" 선택 가능.

2.  **메뉴 관리 (Menu Management)**
    *   **내 리스트:** 사용자가 선호하는 메뉴 추가/삭제/수정.
    *   **프리셋 제공:** '한식', '중식', '일식', '분식', '비오는 날' 등 상황별 기본 리스트 제공.
    *   **블랙리스트:** 절대 먹기 싫은 메뉴 제외 기능.

3.  **공유하기 (Share)**
    *   결과 화면을 캡처하듯 이미지로 공유 ("오늘의 점심 운세는 제육볶음입니다!")

### 3.3 UI/UX 스케치
*   **메인 화면:** 깔끔한 단색 배경(또는 그라데이션), 중앙에 강조된 텍스트와 버튼. 불필요한 배너나 메뉴 제거.
*   **설정 화면:** 바텀 시트(Bottom Sheet) 등을 활용하여 메뉴 리스트를 손쉽게 편집.

## 4. 기대 효과 (Benefits)
*   **시간 절약:** 메뉴 고민 시간을 획기적으로 단축.
*   **소소한 즐거움:** 동료들과 함께 돌려보며 점심시간의 재미 요소 추가.
*   **확장 가능성:** 점심 메뉴뿐만 아니라 '회식 장소', '벌칙 당첨자', '오늘의 할 일' 등으로 컨텐츠 확장 용이.

## 5. 기술적 고려사항 (Technical Considerations)
*   **Platfrom:** Android (Jetpack Compose)
*   **Data Storage:**
    *   **Room Database:** 메뉴 데이터의 저장 및 관리 (CRUD).
    *   **DataStore:** 앱 설정 (애니메이션 속도, 사운드 On/Off) 저장.
*   **Animation:** `Animatable` 또는 `InfiniteTransition`을 활용한 감속 애니메이션 구현. (자연스러운 물리 효과가 중요)
*   **Architecture:** MVVM 패턴 (ViewModel에서 뽑기 로직 처리, UI는 상태만 관찰).
