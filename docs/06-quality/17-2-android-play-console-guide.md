## Play Console 시작하기

### 🎯 Google Play Console이란?

**Google Play Console**은 Android 앱을 Google Play Store에 배포하고 관리하는 플랫폼입니다.

```
개발자 → Play Console → Google Play Store → 사용자
```

### 📋 사전 준비사항

#### 1. Google 계정
- Gmail 계정 필요
- 비즈니스용 계정 권장

#### 2. 개발자 등록비
- **$25 USD** (일회성 결제)
- 신용카드 또는 체크카드 필요

#### 3. 준비물
- 서명된 AAB 파일
- 앱 아이콘 (512x512 PNG)
- 스크린샷 (최소 2개)
- 개인정보처리방침 URL

### 🚀 개발자 계정 생성

#### 1단계: Play Console 접속

1. [Google Play Console](https://play.google.com/console) 접속
2. Google 계정으로 로그인

#### 2단계: 개발자 계정 등록

```
1. "계정 만들기" 클릭
2. 계정 유형 선택:
   - 개인 개발자
   - 조직 (회사)
3. 개발자 정보 입력:
   - 개발자 이름 (공개됨)
   - 이메일 주소
   - 전화번호
4. 결제 정보 입력 ($25)
5. 개발자 배포 계약 동의
```

> [!TIP]
> **개발자 이름은 신중하게 선택하세요!** Play Store에 공개되며 나중에 변경하기 어렵습니다.

#### 3단계: 본인 인증

```
Google이 본인 인증을 요구할 수 있습니다:
- 신분증 사본
- 사업자 등록증 (조직인 경우)
- 주소 증명서
```

## 앱 생성 및 기본 설정

### 📱 새 앱 만들기

#### 1단계: 앱 생성

```
1. Play Console 홈 → "모든 앱" → "앱 만들기"
2. 앱 세부정보 입력:
   - 앱 이름 (최대 50자)
   - 기본 언어
   - 앱 또는 게임 선택
   - 무료 또는 유료 선택
3. 선언문 체크:
   ✅ 개발자 프로그램 정책 준수
   ✅ 미국 수출법 준수
4. "앱 만들기" 클릭
```

**앱 이름 작성 팁**:
```kotlin
// ❌ 나쁜 예
"앱"
"MyApp123"
"최고의앱최고의앱최고의앱"

// ✅ 좋은 예
"할 일 관리 - Todo List"
"영어 단어장 - Vocabulary"
"운동 기록 - Fitness Tracker"
```

#### 2단계: 앱 액세스 권한

```
앱이 로그인을 요구하는 경우:
1. "앱 액세스 권한" 섹션
2. "모든 기능이 제한 없이 제공됨" 또는
   "일부 기능이 제한됨" 선택
3. 제한된 경우 테스트 계정 제공:
   - 이메일: test@example.com
   - 비밀번호: TestPassword123
```

#### 3단계: 광고 포함 여부

```
1. "광고" 섹션
2. 광고 포함 여부 선택:
   - 예: 앱에 광고가 표시됨
   - 아니요: 광고 없음
```

## 스토어 등록정보

### 🎨 앱 세부정보

#### 1. 앱 이름 및 설명

**간단한 설명** (최대 80자):
```
앱의 핵심 기능을 한 줄로 요약

예시:
"할 일을 쉽고 빠르게 관리하는 투두 리스트 앱"
"매일 영어 단어를 학습하고 복습하는 단어장"
```

**자세한 설명** (최대 4,000자):
```
1. 앱 소개 (2-3문장)
2. 주요 기능 (불릿 포인트)
3. 사용 방법
4. 고유한 장점
5. 연락처 정보

예시:
할 일 관리 - Todo List

간단하고 직관적인 할 일 관리 앱입니다. 
복잡한 기능 없이 필요한 것만 담았습니다.

주요 기능:
• 할 일 추가 및 완료 체크
• 카테고리별 분류
• 알림 설정
• 다크 모드 지원
• 위젯 지원

사용 방법:
1. + 버튼을 눌러 할 일 추가
2. 완료하면 체크
3. 스와이프로 삭제

왜 이 앱을 사용해야 하나요?
• 광고 없음
• 무료
• 빠르고 가벼움
• 개인정보 수집 안 함

문의: support@example.com
```

#### 2. 그래픽 에셋

**앱 아이콘** (512x512 PNG, 32비트):
```
요구사항:
- 크기: 512x512 픽셀
- 형식: PNG (32비트)
- 투명 배경 가능
- 모서리 라운드 자동 적용

디자인 팁:
✅ 단순하고 명확한 디자인
✅ 브랜드 색상 사용
✅ 텍스트 최소화
❌ 너무 복잡한 디테일
❌ 작은 텍스트
```

**기능 그래픽** (1024x500 JPG/PNG):
```
요구사항:
- 크기: 1024x500 픽셀
- 형식: JPG 또는 PNG
- 최대 용량: 1MB

용도:
- Play Store 상단 배너
- 프로모션 자료

디자인 팁:
✅ 앱의 주요 기능 강조
✅ 브랜드 아이덴티티 반영
✅ 고품질 이미지
❌ 텍스트 과다 사용
```

**스크린샷** (최소 2개, 권장 8개):
```
휴대전화 스크린샷:
- 최소 크기: 320픽셀
- 최대 크기: 3840픽셀
- 종횡비: 16:9 ~ 2:1

태블릿 스크린샷 (선택):
- 7인치 태블릿
- 10인치 태블릿

스크린샷 작성 팁:
1. 주요 화면 캡처
2. 핵심 기능 강조
3. 깔끔한 UI 상태
4. 텍스트 오버레이로 설명 추가
5. 일관된 스타일 유지

예시 순서:
1. 메인 화면
2. 주요 기능 1
3. 주요 기능 2
4. 설정 화면
5. 다크 모드
```

#### 3. 분류

**앱 카테고리**:
```
선택 가능한 카테고리:
- 도구
- 생산성
- 교육
- 건강 및 피트니스
- 소셜
- 엔터테인먼트
등...

태그 (최대 5개):
앱을 설명하는 키워드 선택
```

## 앱 콘텐츠

### 🔞 콘텐츠 등급

#### 1단계: 설문지 작성

```
1. "콘텐츠 등급" → "설문지 시작"
2. 이메일 주소 입력
3. 앱 카테고리 선택
4. 질문에 답변:
   - 폭력성
   - 성적 콘텐츠
   - 언어
   - 약물
   - 도박
   등...
```

**질문 예시**:
```
Q: 앱에 폭력적인 콘텐츠가 포함되어 있나요?
A: 아니요

Q: 앱에서 사용자 간 소통이 가능한가요?
A: 예 (채팅 기능이 있는 경우)

Q: 앱에 광고가 포함되어 있나요?
A: 예/아니요
```

#### 2단계: 등급 확인

```
설문지 제출 후 등급 부여:
- 전체 이용가
- 12세 이용가
- 15세 이용가
- 청소년 이용불가

각 국가별 등급:
- ESRB (미국)
- PEGI (유럽)
- GRAC (한국)
등...
```

### 🎯 대상 고객

```
1. "대상 고객 및 콘텐츠" 섹션
2. 대상 연령층 선택:
   - 13세 미만만 대상
   - 13세 미만 및 그 이상 대상
   - 13세 이상만 대상

3. 어린이 대상 앱인 경우:
   - COPPA (미국 아동 온라인 개인정보 보호법) 준수
   - 광고 제한
   - 데이터 수집 제한
```

### 🔒 개인정보처리방침

```
1. "개인정보처리방침" 섹션
2. URL 입력 (필수)

개인정보처리방침 작성 팁:
✅ 수집하는 정보 명시
✅ 정보 사용 목적
✅ 정보 보관 기간
✅ 제3자 제공 여부
✅ 사용자 권리
✅ 연락처

예시 URL:
https://yourapp.com/privacy-policy
```

### 📊 데이터 보안

```
1. "데이터 보안" 섹션
2. 수집하는 데이터 유형 선택:
   - 위치
   - 개인 정보
   - 금융 정보
   - 사진 및 동영상
   - 기기 또는 기타 ID
   등...

3. 각 데이터 유형별로:
   - 수집 여부
   - 공유 여부
   - 선택 가능 여부
   - 암호화 여부
```

## 테스트 및 출시

### 🧪 내부 테스트

**목적**: 소규모 팀 내부 테스트

```
1. "테스트" → "내부 테스트" → "새 출시 만들기"
2. AAB 업로드
3. 출시 이름 입력 (예: "v1.0.0-alpha")
4. 출시 노트 작성
5. "검토 시작" → "출시 시작"

테스터 추가:
1. "테스터" 탭
2. 이메일 목록 생성
3. 테스터 이메일 추가 (최대 100명)
4. 저장

테스터에게 공유:
- 테스트 링크 복사
- 이메일로 전송
- 테스터가 링크를 통해 앱 다운로드
```

### 🔒 비공개 테스트

**목적**: 더 큰 그룹으로 테스트 (최대 수천 명)

```
1. "테스트" → "비공개 테스트" → "새 출시 만들기"
2. AAB 업로드
3. 출시 노트 작성
4. 테스터 그룹 생성:
   - 이메일 목록
   - Google 그룹
5. "검토 시작" → "출시 시작"

테스트 기간:
- 최소 14일 권장
- 버그 수정 및 피드백 반영
```

### 🌐 공개 테스트

**목적**: 누구나 참여 가능한 공개 베타

```
1. "테스트" → "공개 테스트" → "새 출시 만들기"
2. AAB 업로드
3. 국가 선택 (전체 또는 특정 국가)
4. 참여자 수 제한 (선택)
5. "검토 시작" → "출시 시작"

장점:
- 대규모 사용자 피드백
- 실제 환경 테스트
- 마케팅 효과
```

### 🚀 프로덕션 출시

#### 1단계: 출시 준비

```
모든 필수 항목 완료 확인:
✅ 스토어 등록정보
✅ 콘텐츠 등급
✅ 대상 고객
✅ 개인정보처리방침
✅ 데이터 보안
✅ 앱 콘텐츠
```

#### 2단계: 출시 만들기

```
1. "프로덕션" → "새 출시 만들기"
2. AAB 업로드
3. 출시 이름 입력 (예: "v1.0.0")
4. 출시 노트 작성:

예시:
버전 1.0.0의 새로운 기능:
• 할 일 추가 및 관리
• 카테고리별 분류
• 알림 기능
• 다크 모드 지원

5. "검토 시작" 클릭
```

#### 3단계: 국가 및 지역 선택

```
1. "국가/지역" 탭
2. 출시할 국가 선택:
   - 모든 국가
   - 특정 국가만
   
팁:
- 처음에는 소수 국가로 시작
- 안정화 후 점진적 확대
```

#### 4단계: 단계적 출시 (권장)

```
1. "출시 개요" → "단계적 출시 관리"
2. 출시 비율 선택:
   - 1% (테스트)
   - 5%
   - 10%
   - 20%
   - 50%
   - 100% (전체)

단계적 출시 전략:
Day 1: 1% → 문제 모니터링
Day 3: 5% → 안정성 확인
Day 5: 20% → 서버 부하 확인
Day 7: 50%
Day 10: 100% → 전체 출시
```

#### 5단계: Google 검토

```
검토 프로세스:
1. Google이 앱 검토 (보통 1-3일)
2. 정책 위반 확인
3. 승인 또는 거부

거부 사유:
- 정책 위반
- 저작권 침해
- 악성 코드
- 오해의 소지가 있는 설명
```

## 출시 후 관리

### 📊 통계 및 분석

```
Play Console 대시보드:
1. "통계" 탭
2. 확인 가능한 지표:
   - 설치 수
   - 제거 수
   - 평점 및 리뷰
   - 충돌 및 ANR
   - 국가별 통계
   - 기기별 통계
```

### ⭐ 평점 및 리뷰 관리

```kotlin
/**
 * 리뷰 응답 베스트 프랙티스
 */

// ✅ 좋은 응답
"소중한 피드백 감사합니다! 
다음 업데이트에서 요청하신 기능을 추가하겠습니다."

"불편을 드려 죄송합니다. 
해당 문제는 v1.1.0에서 수정되었습니다. 
업데이트 부탁드립니다."

// ❌ 나쁜 응답
"그건 버그가 아닙니다."
"다른 앱 쓰세요."
(응답 없음)
```

**리뷰 관리 팁**:
```
1. 모든 리뷰에 응답 (가능한 한)
2. 부정적 리뷰에 특히 신경 쓰기
3. 24시간 내 응답 목표
4. 정중하고 전문적인 태도
5. 구체적인 해결 방안 제시
```

### 🔄 앱 업데이트

```
1. "프로덕션" → "새 출시 만들기"
2. 새 AAB 업로드
3. versionCode 증가 확인
4. 출시 노트 작성:

예시:
버전 1.1.0 업데이트:

새로운 기능:
• 위젯 추가
• 백업 및 복원 기능

개선 사항:
• 앱 시작 속도 30% 향상
```
